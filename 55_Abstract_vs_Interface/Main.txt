| Feature                          | **Abstract Class**                                                               | **Interface**                                                                                                       |
| -------------------------------- | -------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------- |
| **Keyword**                      | `abstract class`                                                                 | `interface`                                                                                                         |
| **Purpose**                      | To provide **partial abstraction** (some implementation + some abstract methods) | To provide **full abstraction** (only method declarations)                                                          |
| **Method Type**                  | Can have **abstract** and **non-abstract (implemented)** methods                 | Before Java 8 → only abstract methods <br> After Java 8 → can have **default**, **static**, and **private** methods |
| **Variables**                    | Can have **instance variables**, static variables, and constants                 | All variables are **public, static, and final** (constants) by default                                              |
| **Constructors**                 | ✅ Can have constructors                                                          | ❌ Cannot have constructors                                                                                          |
| **Access Modifiers for Methods** | Can be `private`, `protected`, or `public`                                       | All methods are `public` by default                                                                                 |
| **Multiple Inheritance**         | ❌ A class can extend **only one abstract class**                                 | ✅ A class can **implement multiple interfaces**                                                                     |
| **When to Use**                  | When classes share **common base functionality**                                 | When you need to **define a contract** that multiple unrelated classes can follow                                   |
| **Inheritance Type**             | `extends` keyword                                                                | `implements` keyword                                                                                                |
| **Abstract Method Declaration**  | `abstract void show();`                                                          | `void show();` (no `abstract` keyword needed)                                                                       |
| **Example Use Case**             | Base class for related objects (e.g., `Vehicle`, `Shape`)                        | Common behavior for unrelated objects (e.g., `Drawable`, `Comparable`)                                              |
